HARI API (1.0.0)

Download OpenAPI specification:

Documentation des endpoints HTTP exposés par l'application HARI (Next.js App Router).

L'application n'expose volontairement que deux familles d'endpoints :

  • /api/auth/* — Auth.js v5 (authentification par identifiants, sessions JWT).
  • /api/chat — point d'entrée unique de l'assistant IA conversationnel, qui orchestre lui-même les "outils" RH (lib/ai/tools.ts) listés en §x-ai-tools ci-dessous.

Toute autre donnée (annuaire, congés, paie, etc.) est servie par des composants serveur Next.js qui appellent directement lib/hr.ts — il n'y a pas d'API REST CRUD séparée à documenter, par conception (cf. AGENTS.md, "Authorization is always server-side").

Chat

Assistant IA conversationnel RH (streaming, appels d'outils RBAC)

Envoyer un message à l'assistant IA HARI

Point d'entrée unique du chat. Authentification obligatoire (401 sinon, vérifié avant tout appel modèle — voir tests/api.chat.test.ts). La réponse est un flux SSE (Vercel AI SDK UI Message Stream) contenant le texte généré, le raisonnement, et les appels d'outils RH exécutés sous garde RBAC (withPermission, voir x-ai-tools ci-dessous). Un outil refusé ne lève pas d'erreur HTTP : il renvoie { denied: true } dans le flux, rendu comme une carte "accès refusé" côté client.
